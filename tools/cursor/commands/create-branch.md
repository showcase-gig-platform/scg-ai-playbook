# create-branch

現在の変更内容を確認し、適切なブランチ名を提案してブランチを作成するコマンドです。

## ブランチ名のプレフィックス規則

- **新機能**: `feat/` - 新機能の追加
- **修正**: `fix/` - バグ修正や不具合の修正
- **リリース用 PR**: `release/` - リリース準備用のブランチ
- **大きい epic 系**: `epic/` - 大きな機能や複数の機能を含む epic

## 実行手順

1. **現在の Git の状態を確認**

   ```bash
   git status
   ```

   変更されたファイルとステージング状態を確認

2. **変更内容の詳細を確認**（必要に応じて）

   ```bash
   git diff
   ```

   またはステージング済みの変更を確認

   ```bash
   git diff --cached
   ```

3. **変更内容を分析してブランチの種類を判定**

   - 新規ファイルの追加や新機能の実装 → `feat/`
   - 既存コードのバグ修正や不具合対応 → `fix/`
   - 複数の機能にまたがる大きな変更、複数のコミットを含む → `epic/`
   - バージョン番号の更新、リリースノート、リリース関連の変更 → `release/`

4. **ブランチ名を提案**

   - 変更されたファイル名、ディレクトリ名、機能名から推測
   - プレフィックス + 機能名や修正内容を表すスラッシュ区切りの名前
   - 例:
     - `feat/add-user-authentication`
     - `fix/resolve-payment-error`
     - `release/v1.2.0`
     - `epic/mobile-order-system`
   - ブランチ名は小文字、ハイフン区切りを推奨

5. **ユーザーに確認**

   - 提案したブランチ名を提示
   - ユーザーが承認するか、別の名前を指定するか確認

6. **ブランチを作成**
   ```bash
   git checkout -b [ブランチ名]
   ```
   または、現在のブランチから分岐する場合
   ```bash
   git checkout -b [ブランチ名] [ベースブランチ]
   ```

## 注意事項

- 既にブランチが存在する場合は、エラーを表示して確認を求める
- 変更内容が不明確な場合は、ユーザーに確認してからブランチ名を決定する
- 現在のブランチが main や master などのメインブランチの場合は、警告を表示する
- 未コミットの変更がある場合は、その旨を伝える
