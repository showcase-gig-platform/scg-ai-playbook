# create-pr

## 目的

GitHub MCPを使用して、プロジェクトのPRテンプレートに従ったPull Requestを作成します。

## 前提条件

- GitHub MCPが設定されている（`setup-github-mcp`コマンドで設定済み）
- 作業ブランチで変更がコミット済み
- リモートにプッシュ済み

## 実行内容

このコマンドを実行すると、以下の処理を自動で行います：

### 1. 事前確認（自動実行）

以下のコマンドを実行して現在の状態を確認します：

```bash
git status
git branch --show-current
git log main..HEAD --oneline
git diff main --stat
git remote get-url origin
```

確認項目：
- 現在のブランチ名
- 未コミットの変更の有無
- リモートへのプッシュ状態
- 変更されたファイル一覧
- コミット履歴

### 2. PR情報の自動生成と収集

#### 手動入力が必要な項目（2つ）

1. **ベースブランチ**: マージ先のブランチ（デフォルト: `main`）
2. **チケットID / リンク**: ClickUp、JIRA、GitHub Issue等のリンク

#### 自動生成される項目

以下の項目は変更内容から自動的に生成されます：

| 項目 | 自動生成ロジック |
|------|-----------------|
| **PRタイトル** | ブランチ名から推測（例: `feat/add-login` → `feat: add-login`）またはコミットメッセージの要約 |
| **PRの目的** | コミットメッセージの内容を整理して生成 |
| **影響範囲** | `git diff main --stat` の結果から変更ファイル一覧を取得 |
| **破壊的変更点** | 変更ファイルを分析（`schema/*.sql`, `proto/*.proto` の変更を検出）|
| **動作確認項目** | テストファイル（`*_test.go`）の変更有無から自動生成 |
| **参考リンク** | 基本「なし」（必要に応じてユーザーが追加可能）|

#### 破壊的変更の自動検出ルール

以下のファイルが変更されている場合、破壊的変更として警告：
- `schema/*.sql` - DBスキーマ変更
- `proto/*.proto` - API定義変更
- `app/presentation/controller/**` - エンドポイント変更

### 3. GitHub MCPでPR作成

`user-github-create_pull_request`ツールを使用してPRを作成します。

## PRテンプレート形式

```markdown
<!-- I want to review in Japanese. -->
## チケットID もしくは リンク
{チケットIDまたはリンク}

## このPRで解決/修正する目的
{修正の背景と目的}

## 影響範囲
### 影響のある箇所
{影響を受けるファイル、エンドポイント、機能など}

### 破壊的変更点
{破壊的変更がある場合は記載、なければ「なし」}

### 注意事項
破壊的なDBスキーマの変更がある場合は２段階でリリースを行うこと

## 動作確認項目
{テスト済みの項目をチェックリストで記載}

## (参考リンク)
{参考になるリンクがあれば記載、なければ「なし」}

# レビュー可能チェック
- [x] すべての項目を記入しました
<!-- I want to review in Japanese. -->
```

## 使用方法

### コマンド実行

```
/create-pr
```

### 実行フロー

1. **事前確認の実行**
   - ブランチ状態、変更内容を自動で確認
   - 未コミットの変更があれば警告を表示
   - リモートへのプッシュ状態を確認

2. **ベースブランチとチケットリンクの入力**
   - ベースブランチ（デフォルト: `main`、Enterで確定）
   - チケットリンク（ClickUp、JIRA、GitHub Issue等）

3. **PR内容の自動生成**
   - PRタイトル: ブランチ名・コミットメッセージから生成
   - PRの目的: コミットメッセージから生成
   - 影響範囲: 変更ファイル一覧から自動取得
   - 破壊的変更点: schema/proto/controller変更を自動検出
   - 動作確認項目: テストファイル変更から自動生成

4. **PR内容のプレビューと確認**
   - 作成されるPRの内容をプレビュー表示
   - 修正が必要な場合はユーザーが編集可能
   - 「OK」で確定

5. **PR作成と結果表示**
   - GitHub MCPでPRを作成
   - 作成されたPRのURLを表示

## 自動生成される情報

以下の情報は自動的に取得・生成されます：

### Git情報（自動取得）
- リポジトリのowner/repo（git remoteから取得）
- ヘッドブランチ（現在のブランチ）
- 変更されたファイル一覧
- コミット履歴

### 手動入力
- ベースブランチ（デフォルト: `main`）
- チケットリンク

### PR本文（自動生成）
- **PRタイトル**: ブランチ名またはコミットメッセージから生成
- **PRの目的**: コミットメッセージを整理して生成
- **影響範囲**: 変更ファイルをカテゴリ分けして表示
- **破壊的変更点**: schema/proto/controllerの変更を自動検出
- **動作確認項目**: テストファイルの変更からチェックリスト生成
- **参考リンク**: デフォルトは「なし」

## 注意事項

- PRを作成する前に、必ずローカルの変更をコミット＆プッシュしてください
- ブランチ名は適切な命名規則に従ってください
- 破壊的なDBスキーマ変更がある場合は、2段階リリースを検討してください

## エラー時の対応

### ブランチがリモートにない場合

```bash
git push -u origin <branch-name>
```

### 未コミットの変更がある場合

```bash
git add .
git commit -m "コミットメッセージ"
git push
```

### GitHub MCPが接続されていない場合

`/setup-github-mcp` コマンドを実行してGitHub MCPを設定してください。

### リモートURLが取得できない場合

```bash
git remote -v
```

でリモートが設定されているか確認してください。
